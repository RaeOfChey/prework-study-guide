HTML

HTML Elements and Structure

  • HTML is used to create the structure of a webpage, and it consists of elements that work together to provide content and the basic framework for all web pages.
  • Common elements used to add content to pages include text elements, image elements, header elements, and list elements.
  • Elements can be used to divide and describe content for browsers and other developers.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

HTML Elements Types

  • Meta-related elements contain information about the page used by browsers.
  • Content division elements are used to structure content on the page (e.g., <div>, <span>).
  • Semantic elements describe the role of the content, making it easier to read by browsers and developers (e.g., <header>, <footer>, <article>).

Metadata and the <head> Element

The <head> element contains metadata for the webpage, such as accessibility and performance information, which helps the browser render the page correctly. Common tags inside <head> include:
  • <title>: Defines the title of the webpage, essential for SEO.
  • <style>: Defines CSS styles.
  • <link>: Connects external resources, such as stylesheets.
  • <meta>: Provides metadata like character encoding, description, and keywords.

HTML Attributes

HTML attributes are defined in the opening tag of an element (e.g., <img src="..." alt="...">). Common attributes include:
  • <src> for specifying the source of an image in the <img> tag.
  • <alt> for providing an alternate text description for images.

Parent and Child Elements

  • Parent and child elements refer to the relationship between elements, where nested elements inside a parent are referred to as children.
  • Example: Elements nested inside the <body> are child elements of <body>.

Tags vs Elements

A tag refers to the syntax inside angle brackets (e.g., <head>), while an element refers to the entire structure, including its opening and closing tags (e.g., <head>...</head>).

List and Image Elements

  • The <ul>, <ol>, and <li> elements are used to create unordered (bulleted) and ordered (numbered) lists.
  • The <img> element defines an image on a webpage, and attributes such as src (source) and alt (alternative text) are important for rendering and accessibility.

Anchor and Break Elements

  • The <a> (anchor) element is used to create hyperlinks that link to other webpages or sections of the same page. It uses the href attribute to define the link’s destination.
  • The <br> element creates a line break, adding a space between blocks of content.

CSS

CSS Overview

  • CSS stands for Cascading Style Sheets.
  • CSS is used to style and layout web pages, controlling the visual appearance of content.
  • CSS can be applied inline, internally (within the <style> tag in HTML), or externally via a linked .css file.

CSS Syntax

  • CSS rules are made up of selectors and declarations.
  • Selectors target HTML elements you want to style. Example: p { color: red; } styles all <p> elements to have red text.
  • Declarations consist of a property and a value, contained within curly braces {}. Example: color: red; changes text color to red.
  • Multiple declarations can be grouped for a selector, separated by semicolons. Example: p { color: red; font-size: 16px; }.

Common CSS Selectors

  • Type Selector targets all instances of a specific HTML element (e.g., h1, p).
  • Class Selector targets elements with a specific class attribute. Use a period (.) before the class name (e.g., .className).
  • ID Selector targets a single element with a specific ID attribute. Use a hash (#) before the ID (e.g., #idName).
  • Universal Selector targets all elements (e.g., *).
  • Descendant Selector targets elements nested within a specified element (e.g., div p targets all <p> inside a <div>).

CSS Box Model

Every HTML element is represented as a box in CSS. The box model consists of:
  • Content: The actual content of the element, such as text or images.
  • Padding: Clears space inside the element, around the content.
  • Border: Surrounds the padding (if any) and content.
  • Margin: Clears space outside the element, around the border.

CSS Positioning

  • Static Positioning is the default positioning; elements are placed according to the normal document flow.
  • Relative Positioning is the position an element relative to its normal position.
  • Absolute Positioning is the position an element relative to its nearest positioned ancestor.
  • Fixed Positioning is the position an element relative to the browser window, so it stays in the same place even if the page is scrolled.

CSS Flexbox

A layout model for arranging elements in rows or columns with the ability to expand and shrink. Main properties:
  • display: flex;: Applies Flexbox to the container.
  • flex-direction: Defines the direction of the flex items (row, column).
  • justify-content: Aligns items along the main axis.
  • align-items: Aligns items along the cross axis.

CSS Grid

A two-dimensional layout system for creating more complex layouts. Main properties:
  • display: grid;: Applies Grid to the container.
  • grid-template-columns: Defines the number and size of columns.
  • grid-template-rows: Defines the number and size of rows.

CSS Positioning

  • color: Changes the text color (e.g., color: blue;).
  • background-color: Changes the background color of an element.
  • font-size: Sets the size of the text (e.g., font-size: 16px;).
  • margin: Adds space outside the element's border (e.g., margin: 10px;).
  • padding: Adds space inside the element's border (e.g., padding: 20px;).
  • border: Adds a border around the element (e.g., border: 1px solid black;).

Media Queries

Media queries allow you to apply different styles for different screen sizes or devices. Example: @media (max-width: 600px) { ... } applies styles when the screen width is 600px or smaller.

Media Queries

CSS allows the use of custom properties (variables) to reuse values throughout the stylesheet. Example: --main-color: #3498db; defines a variable, and color: var(--main-color); uses it.

Pseudo-classes and Pseudo-elements

  • Pseudo-classes: Define a special state of an element (e.g., :hover, :focus).
  • Pseudo-elements: Style specific parts of an element (e.g., ::before, ::after).

Git

Git Overview

Git is a version control system used to track changes in source code during software development. It allows multiple developers to work on a project simultaneously, managing changes and maintaining version history. GitHub, GitLab, and Bitbucket are popular platforms for hosting Git repositories.

CSS Positioning

  • git init: Initializes a new Git repository in the current directory.
  • git clone: Clones (copies) an existing repository from a remote server to your local machine.
  • git add: Adds files to the staging area, preparing them to be committed. Example: git add filename or git add . to add all changes.
  • git commit: Records changes in the repository with a message. Example: git commit -m "Initial commit".
  • git status: Displays the state of the working directory and staging area, showing which files have been modified or are staged for commit.
  • git push: Uploads local commits to a remote repository. Example: git push origin main to push to the main branch.
  • git pull: Fetches changes from a remote repository and merges them into your local repository.
  • git branch: Lists branches or creates a new branch. Example: git branch new-feature to create a new branch.
  • git checkout: Switches branches or restores files. Example: git checkout new-feature to switch to the new-feature branch.
  • git merge: Merges changes from one branch into another. Example: git merge feature-branch merges the feature-branch into the current branch.

CSS Positioning

  • Branches are used to work on different features or bug fixes in isolation.
  • Main Branch: Usually called main or master, it is the default and stable branch.
  • Feature Branches: Created for new features or fixes. Merged back into the main branch once work is complete.

CSS Positioning

  • Create a branch for a new feature or fix: git branch feature-name.
  • Switch to that branch: git checkout feature-name.
  • Make changes to files, then add and commit those changes.
  • Once done, merge the feature branch back into main and push changes to the remote repository.

Javascript

JavaScript Overview

JavaScript (JS) is a programming language used to create dynamic and interactive content on web pages. JavaScript can be used to manipulate HTML and CSS, allowing developers to create interactive websites.

JavaScript Syntax

Variables: Store data values and can be declared using var, let, or const. Example: let name = "Cheyenna"; Data Types: Common data types include:
  • String: "Hello, World!"
  • Number: 42
  • Boolean: true or false
  • Array: let arr = [1, 2, 3];
  • Object: let person = { name: "Cheyenna", age: 25 };

JavaScript Functions

Functions are blocks of code designed to perform a task. They can be declared or assigned to variables.
  • Arrow Functions: A shorter syntax for writing functions. Example: const greet = () => console.log("Hello, World!");

JavaScript DOM Manipulation

JavaScript can be used to access and modify HTML elements via the DOM (Document Object Model).
  • document.getElementById(): Selects an element by its ID. Example: document.getElementById("myElement").innerText = "New Text";
  • document.querySelector(): Selects the first element that matches a CSS selector. Example: document.querySelector(".myClass").style.color = "blue";

JavaScript Events

JavaScript responds to user actions like clicks, keypresses, or form submissions through event listeners. addEventListener(): Attaches an event handler to an element.

JavaScript Control Structures

Conditionals are used to perform different actions based on conditions. Loops are repeat blocks of code. There are different types of loops.

  • while loop

JavaScript Objects and Arrays

Objects store data in key-value pairs. Arrays store lists of data.

JavaScript DOM Manipulation

  • let and const: Block-scoped variable declarations.
  • Arrow Functions: Shorter function syntax.
  • Template Literals: Use backticks (`) to include variables in strings. Example: `Hello, ${name}`

Asynchronous JavaScript

Promises handle asynchronous operations. async/await: A modern syntax for writing asynchronous code.
