HTML

HTML (Hypertext Markup Language) is the foundational language used to structure the content on the web. It creates the skeleton for web pages by using various elements, which are enclosed within tags. HTML elements define the different parts of the webpage, such as headers, paragraphs, images, and links.

Elements & Tags

HTML is composed of elements, where each element has an opening tag (<tag>), optional content, and a closing tag (</tag>). Some elements are self-closing, like <img> and <br>.

Common structural elements include:

  • <html> are the root element of an HTML document.
  • <head> contains metadata and links to stylesheets or scripts.
  • <body> contains all the content that is displayed on the web page.

HTML Document Structure

The typical HTML document begins with <!DOCTYPE html> to declare the document as HTML5.

The document contains:

  • <html> is an element, with lang="en" to define the document language.
  • <head> is a section containing metadata and external references like CSS and JS files.
  • <body> is a section for the visible content of the webpage.

Head Element

Contains metadata, which is information not displayed on the page but used by browsers and search engines.

Important elements inside <head>:

  • <meta> defines character sets, viewport settings, and other metadata.
  • <title> defines the page's title (displayed on the browser tab).
  • <link> links external resources, such as CSS files.
  • <script> embeds or links JavaScript files.

Body Element

The <body> contains the content of the webpage. It holds all visible elements such as:

  • Text <p> is for paragraphs, <h1> to <h6> for headings.
  • Images <img> is for embedding images.
  • Lists <ul> is for unordered (bulleted) lists, <ol> for ordered (numbered) lists, <li> for list items.
  • Links <a> is for hyperlinks.

Attributes

HTML elements often have attributes, which provide additional information about the element. Attributes are always specified in the opening tag and follow the format name="value".

Common attributes include:

  • id provides a unique identifier for an element.
  • class Cclassifies elements for styling or scripting purposes.
  • src defines the source file for media like images.
  • alt provides alternative text for images, improving accessibility.
  • href specifies the URL in anchor elements.

Semantic HTML

Semantic elements help make the HTML more understandable and accessible. These elements describe their meaning:

  • <article> represents an independent piece of content.
  • <section> groups related content under a thematic section.
  • <nav> contains navigation links.
  • <footer> defines the footer section, usually at the end of the page.
  • <header> defines introductory or navigational content at the top of the page.

Semantic HTML improves accessibility, SEO, and the readability of code for other developers.

Forms

HTML forms allow users to submit data to a server. The <form> element wraps form controls such as:

    <input> allows user input (text, passwords, checkboxes, radio buttons). <textarea> for multi-line text input. <button> for submitting or resetting a form. <select> creates a dropdown list.

Important attributes for forms:

  • action specifies where the form data should be sent.
  • method defines the HTTP method used to send the data (e.g., GET or POST).

Tables

HTML tables are created with <table> and contain:

  • <tr> are table rows.
  • <th> are table headers.
  • <td> are able data cells.

Tables are useful for displaying structured data, but modern web design encourages using them only for tabular data, not for page layout.

Media Elements

HTML supports embedding various media types like images, videos, and audio:

  • <img> embeds an image.
  • <video> embeds video content.
  • <audio> embeds audio content.

Document Object Model (DOM)

The DOM is a programming interface for web documents. It represents the HTML document as a tree of nodes, where each element and piece of content is a node. JavaScript can manipulate the DOM to update the page dynamically.

Common HTML5 Elements

HTML5 introduced several new elements to enhance the semantics and functionality of web pages:

  • <canvas> is for drawing graphics via JavaScript.
  • <audio> and <video> for embedding media without plugins.
  • <progress> and <meter> for displaying progress bars and measurement gauges.

Accessibility & Best Practices

  • Use semantic HTML to ensure assistive technologies like screen readers can accurately interpret the content.
  • Include <alt> attributes for all images to provide descriptions.
  • Organize the structure with headings in hierarchical order (<h1>, <h2>, etc.) for clarity.

CSS

Cascading Style Sheets (CSS) is a core language used to control the visual presentation of web pages. While HTML provides the structure, CSS allows developers to specify how elements should be styled and positioned on the page, making a website more visually engaging and easier to use. CSS is an essential tool for separating the content of a webpage from its design.

CSS works by selecting HTML elements and applying styles such as colors, fonts, layouts, and spacing. Styles are applied through rules written in a CSS file or embedded directly in HTML using the <style> tag.

CSS Selectors

CSS selectors define which HTML elements you want to style. There are various types of selectors:

  • Element Selector targets specific HTML elements, such as h1 or p.
  • Class Selector targets elements with a specific class attribute, using a period (e.g., .example-class).
  • ID Selector argets a single element with a unique ID, using a hash symbol (e.g., #example-id).
  • Attribute Selector targets elements based on attributes (e.g., input[type="text"]).
  • Universal Selector targets all elements on the page (e.g., *).
  • Descendant Selector targets elements that are descendants of another element (e.g., div p for all p tags inside a div).

CSS Properties

There are numerous CSS properties that define how elements appear on the page:

  • Color and Background controls the text color (color), background color (background-color), and background images (>background-image).
  • Font and Text sets font family (font-family), size (font-size), weight (font-weight), and spacing properties (letter-spacing, line-height).
  • Box Model defines the layout of elements using margin, padding, border, and width/height to control spacing around and inside elements.
  • Positioning positions elements using properties like position (static, relative, absolute, or fixed), z-index, and top/right/bottom/left to define their placement on the page.
  • Display and Visibility controls how elements are rendered with display (e.g., block, inline, flex, grid) and visibility.

Responsive Design

Responsive design ensures that a webpage looks good on all screen sizes, from desktops to mobile phones. This is achieved using:

  • Media Queries are CSS rules that apply styles based on the width of the viewport or screen (e.g., @media (max-width: 600px)).
  • Flexible Layouts use relative units like percentages or em remof fixed units like pixels.
  • CSS Flexbox is a layout model that arranges elements in flexible rows and columns, allowing for responsive alignment and distribution of space.
  • CSS Grid is a powerful layout system for creating complex, two-dimensional grid-based layouts with rows and columns.

Cascading and Inheritance

CSS follows a cascading order, meaning that styles can be inherited from parent elements and multiple rules can apply to the same element. The cascade determines which styles take precedence:

  • Specificity determines which styles are applied based on how specific the selector is. IDs are more specific than classes, which are more specific than element selectors.
  • Inheritance comes from when CSS properties are inherited from parent elements to child elements, such as font styles and text colors.
  • Importance is the !important rule can be used to override other styles, but should be used sparingly.

External vs. Internal Styles

CSS can be applied to a webpage in several ways:

  • Inline CSS are styles applied directly within an HTML element using the style attribute (e.g., h1 style="color: blue";).
  • Internal CSS are styles written inside a <style> element within the HTML <head> section.
  • External CSS are separate CSS file linked to the HTML document using the <link> element. This is the preferred method as it keeps the HTML clean and allows for global styling across multiple pages.

CSS Frameworks and Preprocessors

To speed up development, CSS frameworks like Bootstrap provide prebuilt components and responsive design tools. Preprocessors like SASS and LESS allow developers to write more modular and efficient CSS, using features like variables, mixins, and nested rules.

Git

Git is a version control system that tracks changes to project files and helps developers collaborate efficiently. Here's an overview of its essential features and commands:

  • Version Control: Git enables you to keep track of changes made to your project over time, allowing you to revert to previous versions if necessary.
  • Repository (Repo): A repository is the directory where Git stores all the versions of a project, including its files and commit history.
  • Branching: Git allows you to create branches, which are isolated versions of your project. This lets you work on new features or fixes without affecting the main codebase
  • GitHub: GitHub is a popular platform for hosting Git repositories, enabling collaborative work, issue tracking, and code reviews.

Common Git Commands

  • git init - initializes a new Git repository in your project folder.
  • git clone [URL] - copies an existing repository from a remote server (like GitHub) to your local machine.
  • git add [file] - stages changes for commit. You can use git add . to stage all changes.
  • git commit -m "message" - commits the staged changes with a descriptive message.
  • git push - uploads your local changes to the remote repository (e.g., GitHub).
  • git pull - fetches and merges changes from the remote repository to your local copy.
  • git status - isplays the current state of your working directory and staging area.
  • git branch - lists all branches in the repository. You can also create new branches with this command.
  • git checkout [branch] - switches to the specified branch.
  • git merge [branch] - merges changes from the specified branch into the current branch.

Javascript

JavaScript (JS) is a programming language that adds interactivity and logic to web pages. It allows users to interact with the web page by responding to events such as button clicks, form submissions, or other browser actions. JavaScript is executed on the client-side (in the browser) and can dynamically update content without reloading the page.

Key Concepts of JavaScript

Variables store data that can be used and manipulated throughout your code. You can declare variables using var, let, or const.

  • var - function-scoped or globally-scoped variable.
  • let - block-scoped variable.
  • const - block-scoped constant, which cannot be reassigned.

JavaScript supports several data types, including:

  • String - text values, e.g., "Hello, world!"
  • Number - numeric values, e.g., 42
  • Boolean - logical values (true or false)
  • Array - a list of values, e.g., ['HTML', 'CSS', 'JS']
  • Object - key-value pairs representing more complex data, e.g., { name: "Cheyenna", age: 25 }

Functions are reusable blocks of code that perform tasks or calculate values. Functions can take parameters and return values.

Conditional statements allow your program to perform different actions based on different conditions.

  • if/else - basic conditional logic.
  • switch - selects one of many blocks of code to be executed.

Loops repeat a block of code while a condition is true.

  • for - repeats code a specific number of times.
  • while - repeats code as long as a condition is true.

JavaScript can listen for and respond to user events like clicks, keypresses, or mouse movements. Events allow dynamic interaction between users and web pages.

DOM Manipulation

The Document Object Model (DOM) is an interface for interacting with HTML elements using JavaScript. The DOM represents the structure of a webpage, allowing you to dynamically change the content, structure, and style of the page.

  • Selecting Elements - JavaScript can select and manipulate HTML elements based on their ID, class, tag, or other attributes.
  • Modifying Elements - JavaScript allows you to change the content or style of an element.
  • Creating and Adding Elements - you can create new elements and append them to the DOM.

JavaScript Best Practices

  • Keep it modular- write small, reusable functions to make your code easier to manage and debug.
  • Use meaningful variable names - this makes your code easier to read and understand.
  • Error handling - use try/catch blocks to handle errors gracefully.
  • Comment your code - comments help explain what your code is doing.
